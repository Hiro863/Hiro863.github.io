summ_re.process.align_whisper.align_whisper(in_file: str | Path, out_file: str | Path, ipu_file: str | Path, min_length: float = 0.01, tier_name: str = 'transcription', julius_limit: int = 100, keep_words: bool = False, remove_hallu_regex: bool = True) → None#

Compare IPU annotation and timestamped Whisper annotation If the word-level timestamps (partially) overlap with IPU interval add that word to the IPU interval to create a SPPAS compatible transcription file, where annotation corresponds to utterances found in that IPU

Parameters:

  • in_file (Union[str, Path]) – path to the Whisper timestamped annotation (in CSV format)

  • out_file (Union[str, Path]) – path to save output annotation (in CSV format)

  • ipu_file (Union[str, Path]) – path to the IPU annotation (in CSV format)

  • min_length (float) – minimum length for the annotation intervals to keep, default = 0.0 s

  • tier_name (str) – name of the tier, default = ‘transcription’ (this is required for Julius based alignment later

  • julius_limit (int) – Julius accepts tokens up to 150. Set this parameter to limit the number of tokens in a single

  • keep_words (bool) – if true keep word level representation, otherwise concatenate words into IPU level phrases

  • remove_hallu_regex (bool) – if true, removes hallucination markers [*]

Returns:

None

Return type:

None

IPU (it will split them up to multiple IPUs). Note that due to SPPAS wrapper which may change the final number of tokens slightly, setting this value to 150 will still cause an error. The default value set here, 100, seems to work.

Create an IPU annotation using SPPAS SearchIPUs for the file in_file and save it to out_file

Takes searchIPU parameters min_sil, min_ipu, shift_start and shift_end

  • min_sil (float) – minimum duration of a silence

  • min_ipu (float) – minimum duration of an IPU

  • shift_start (float) – start boundary shift value

  • shift_end (float) – end boundary shift value

  • min_n_ipus (int) – minimum number of IPUs to be found to be considered ‘reasonable’

  • min_mean_duration (int) – minimum duration of mean IPU duration to be considered ‘reasonable’

  • manual_thresholds (Union[None, str, Path]) – path to manual annotation if exists, else None

  • rms (Union[Non, int]) – RMS value to give to the SPPAS

summ_re.scripts.overlap.by_type.detect(in_dir: str | Path, out_dir: str | Path, ext: str, include_backchannel: bool = False) → None#

Detect two types of overlaps: full overlap and partial overlap

Full overlap Speaker 1 and 2 fully overlap when speech of one is fully inside speech of another

e.g.

|------------------------------- SPEAKER_1 -------------------------------|

|---- SPEAKER_2 ----|

|--- full overlap --|

Output file names are in the format <first speaker ID>-<second speaker ID>.<csv/TextGrid> Output file is a SPPAS compatible annotation file with annotations corresponding to all SPEAKER_2’s speech that fall within SPEAKER_1’s speech Annotations are marked speech or backchannel

Partial overlap Speaker 1 and 2 partially overlap if either speaker 1 starts speaking before speaker 2 has terminated (start) or conversely, speaker 2 starts speaking before speaker 1 has terminated (end)

e.g. start overlap

|--------------------------- SPEAKER_1 ---------------------------|

– SPEAKER_2 —|

|----- PO ------|

end overlap

|--------------------------- SPEAKER_1 ---------------------------|

|---------------- SPEAKER_2 ----------------|

|-- PO --|

Same naming convention as for full overlap Annotations contain either start for start overlap or end for end overlap

Parameters:

  • in_dir (Union[str, Path]) – path to directory containing output of backchannel annotation files

  • out_dir (Union[str, Path]) – path to directory to save the result

  • ext (str) – extension, csv and TextGrid are allowed

  • include_backchannel (bool) – if true, backchannel intervals are treated like speech

Returns:

None

Return type:

None

summ_re.utils.conversion.du_to_tok(in_file: str | Path, out_file: str | Path, threshold: float)#

Convert DU annotations (inferred from the tokens) to .tok file. The file takes the form of

# new_doc id = IPU_{threshold}:1 1 word - - - - - - - BeginSeg=Yes 2 word - - - - - - - - 3 , - - - - - - - - 4 word - - - - - - - BeginSeg=Yes

# new_doc id = IPU_{threshold}:2 …

, indicates a short pause (< threshold) while new document is generated after longer pauses (> threshold) Short pauses are taken to be the beginning of a DU and is marked by BeginSeg=Yes

Parameters:

  • in_file – paligned annotation file in either CSV or TextGrid

  • out_file – path to output file in .tok

  • threshold – pause threshold in seconds
